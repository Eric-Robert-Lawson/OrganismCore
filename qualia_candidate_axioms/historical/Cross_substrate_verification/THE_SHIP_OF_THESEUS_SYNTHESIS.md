**Pattern persistence:** The maintenance of organizational structure and functional relationships through continuous substrate replacement.

**Key insight:** Pattern is substrate-independent. Same pattern can exist in different substrates.

### 2.2 How Pattern Persistence Works

**The Mechanism (Formalized):**

Let:
- **P(t)** = pattern state at time t
- **S(t)** = substrate state at time t  
- **F(S)** = flux function (substrate replacement rate)
- **M(P,S)** = maintenance function (how pattern uses substrate to persist)

**Pattern persistence occurs when:**

```
P(t+Δt) ≈ P(t)  [pattern approximately stable]

DESPITE:

S(t+Δt) ≠ S(t)  [substrate completely different]

BECAUSE:

M(P(t), S(t)) → S(t+Δt) such that P can continue

AND:

F(S) ≤ M_capacity  [flux rate within maintenance capacity]
```

**In plain language:**

Pattern stays same even though substrate changes, because the pattern itself orchestrates substrate replacement to maintain its own structure.

**This is gap navigation:** Pattern navigates the gap between current substrate state and needed future substrate state to persist.
