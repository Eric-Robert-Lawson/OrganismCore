## PART I: THE TWO DISCOVERIES

### 1.1 RARFL: The Mechanistic Framework

**ORIGIN:**

Developed by Eric Robert Lawson as part of OrganismCore, a universal reasoning substrate enabling objectified, composable reasoning units (RDUs).

**CORE PROBLEM:**

Traditional learning systems assume fixed reward functions. But in nontrivial reasoning spaces, optimal structural invariants (axioms) are unknown in advance. Any static reward function is necessarily incomplete because it cannot encode patterns not yet discovered.

**THE IMPOSSIBILITY:**

**Theorem (from RARFL paper):** Let R be a nontrivial reasoning space. Let F: R → ℝ be a fixed reward function. Then there exists at least one structural invariant α discoverable through exploration that F does not adequately incentivize.

**Proof intuition:** A perfect reward requires knowing all optimal invariants in advance—precisely the information exploration is meant to discover. Static rewards cannot anticipate undiscovered structure.

**THE SOLUTION: RARFL**

**Four-Stage Cycle:**

```
1. EXPLORATION
   Agents navigate R under current reward F
   Generate reasoning trajectories T = {r₁, r₂, ..., rₙ}

2. AXIOM EXTRACTION  
   Identify structural regularities across high-performing trajectories
   Extract candidate axioms A = {α₁, α₂, ..., αₖ}

3. REWARD REFINEMENT
   Update reward function: F' = Ψ(F, A)
   Incorporate discovered axioms into evaluation

4. RE-OPTIMIZATION
   Retrain agents under F'
   Test axiom stability, generate new candidates
   → Return to step 1
```

**KEY INSIGHT:**

The reward function is not an oracle—it's a **hypothesis** about good reasoning. Each RARFL cycle refines this hypothesis by incorporating newly discovered axioms.

**F_{t+1} = Ψ(F_t, A_t)**

Reward and axioms **co-evolve** through their interaction.

## PART II: RARFL AS RELATIONAL PROCESS

### 2.1 The Core RARFL Cycle (Detailed)

**What happens:**
- Agents navigate reasoning space R under current reward F
- Generate trajectories through action/decision sequences
- Accumulate data about what works, what fails

**Relational interpretation:**
- Exploring the gap between current understanding (F) and unknown structure (true axioms)
- Each trajectory tests hypothesis about optimal reasoning
- Failures reveal misalignment between reward and reality

**Example (Chess):**
- Agent plays endgame under reward function
- Some moves lead to faster checkmate (high reward)
- Others prolong game or lose advantage (low reward)
- Pattern: successful moves share structural properties

---

**STAGE 2: AXIOM EXTRACTION**

**What happens:**
- Analyze high-performing trajectories
- Identify recurring structural patterns
- Formalize as candidate axioms α_i
- Build set A = {α₁, α₂, ..., αₖ}

**Relational interpretation:**
- Discovering invariants in the relationship between action and outcome
- Not properties of actions alone (moves are just moves)
- Not properties of outcomes alone (checkmate is just state)
- But relationships: "these structural patterns reliably lead to these outcomes"

**Example (Chess):**
- Across many successful games, notice:
  - King centralization in endgames
  - Rook activity on 7th/8th rank
  - Opposition in king-pawn endings
- These aren't rules of chess (legal moves)
- They're structural invariants of optimal play
- Relationships between position-types and win-probability

---

**STAGE 3: REWARD REFINEMENT**

**What happens:**
- Update reward function: F' = Ψ(F, A)
- Incorporate discovered axioms
- Weight them based on validation
- New reward reflects updated understanding

**Relational interpretation:**
- The reward IS the system's model of good reasoning
- Model was incomplete (couldn't anticipate undiscovered axioms)
- New axioms reveal gap between model and reality
- Update model to close gap
- But new gap will emerge (can't predict future discoveries)

**Example (Chess):**
- Original reward: F(position) = material + basic positional factors
- Discovered axiom: "king centrality critical in endgame"
- Updated reward: F'(position) = material + position + λ·king_centrality
- Now agents optimize for newly recognized invariant

---

**STAGE 4: RE-OPTIMIZATION**

**What happens:**
- Agents retrain under F'
- Generate new trajectories
- Test whether axioms remain stable
- Discover new candidate axioms
- → Loop back to Stage 1

**Relational interpretation:**
- Testing the updated relationship (F' ←→ true structure)
- Stable axioms persist (relationship validated)
- Unstable axioms fade (relationship was spurious)
- New patterns emerge (relationship reveals new gaps)
- The cycle continues

**Example (Chess):**
- Retrain under updated reward
- King centrality axiom validated (still appears in optimal play)
- But new pattern emerges: tempo in endgames
- Extract tempo as new axiom
- Next cycle incorporates both

---

**THE CYCLE AS RELATIONSHIP:**

RARFL is not:

**But:**
- Reward and axioms co-evolving
- Each cycle closes previous gap

- **Asymptotic approach toward optimal structure**

**Neither reward nor axioms exist perfectly in isolation.**
